Representative Input

A reviewer opens the project surface and runs one bounded workflow from a sample input through an exportable or persisted result.

Expected Output

A visible result record containing input summary, action trace, output artifact, error state if any, and claim boundary.

Acceptance Tests

  • Input fixture is explicit and repeatable.
  • Operation under test is specific to this project, not generic portfolio prose.
  • Expected output is inspectable as a file, page state, table, packet, or export.
  • Acceptance criteria are observable and falsifiable.
  • Claim boundary prevents stronger claims than the evidence supports.
  • At least one load-bearing source role is verified or explicitly marked as awaiting human close reading.
  • The workflow records state or produces an exportable output.

Evidence To Capture

  • input fixture
  • operation trace
  • output artifact
  • acceptance-test result
  • source-verification note where applicable
  • claim-boundary note
  • reviewer or outside-use evidence before stronger public claims

Falsifier

The fixture fails if the workflow cannot be repeated, if output is not inspectable, or if the page only describes a system without demonstrating the operation.
