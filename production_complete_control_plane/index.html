Production Standard

Production-complete means the project can be used, checked, reproduced, and defended without relying on private intent or unverified claims. It requires evidence, not only pages.

  1. G1: Purpose and user locked. Public page states what the project is, whom it serves, and what concrete work it performs.
  2. G2: Reachable build surface. A page, workbench, instrument, manuscript packet, or hosted route is reachable from the package.
  3. G3: Instrument or workflow executes. The project can run a representative action, render a result, or expose a completed manuscript/packet workflow.
  4. G4: Non-sample validation. Evidence includes a non-sample run, live case, external review, or manuscript readiness proof, not only a template/demo.
  5. G5: Source and claim traceability. Material claims, methods, and source uses are traceable to project files, sources, or reviewer records.
  6. G6: Regression and falsifier checks. The project has a regression suite, falsifier case, or equivalent challenge condition that prevents easy self-confirmation.
  7. G7: Desktop and mobile evidence. Rendered output is captured on desktop and mobile or the non-web manuscript/software equivalent is documented.
  8. G8: Deployment or reproducibility smoke. The route, packet, script, or local runtime can be opened and its required artifacts can be fetched or reproduced.
  9. G9: Review or signoff evidence. A clearly labeled reviewer, owner, advisor, expert, venue, or Codex technical review note records approval, limits, or remaining defects after inspecting the project result.
  10. G10: No unresolved blocking gaps. The production ledger has no missing blocking gate and no residual evidence slot required for promotion.
