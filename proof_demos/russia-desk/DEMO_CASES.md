# Proof Demo Cases: Russia Security Desk

**Project ID.** `russia-desk`

**Proof tier.** specified-proof-plan-needs-demo

**Public claim.** A Russia security and history desk that combines contemporary intelligence analysis with historical depth.

**Distinctive mechanism.** It separates power, briefing, judgment, and governance subrooms so that Russia work can move between actor analysis, source lineage, historical structure, and claim discipline.

**Component demo cases.** 8

**Claim ceiling.** The page may claim proof-structured components and visible verification targets. It may not claim external validation, production deployment, or completed empirical findings unless the project-specific record states those facts.

## Flagship Demonstrations

### Russia source-to-judgment chain

**Why distinctive.** The proof must show how source families become bounded analytic variables rather than polished commentary.

**Runbook.**

- Choose one visible Russia claim.
- Assign source-family roles.
- Translate sources into analytic variables.
- Run alternative / confidence review.
- State public claim boundary and policy-facing implication.

**Expected evidence.**

- Source family lane
- Analytic variable assignment
- Alternative interpretation note
- Confidence boundary
- Policy-facing output

**Acceptance criteria.**

- The claim is traceable to a source family.
- The variable is explicit enough to be challenged.
- Uncertainty is visible in the public wording.
