# Cognitive Firewalls Defense Room

## Public Pitch

**What it is.** A defense-room variant for explaining the qualitative capstone to an expert reader.

**Distinctive mechanism.** It routes the study through method, evidence packets, coding decisions, reviewer questions, limitations, and post-defense revisions.

**Why it matters.** It is useful only if it keeps the qualitative submission clear and does not blur it with the later quantitative branch.

## Blueprint

### Vision

A defense-room variant for explaining the qualitative capstone to an expert reader.

### Mechanism

It routes the study through method, evidence packets, coding decisions, reviewer questions, limitations, and post-defense revisions.

### Built components

Visible public components: 120 kept Evidence Sections, native navigation, Review Notes, source-use plan, APA lineage, and source lineage claim limit. The public package remains text-focused and does not claim unverified runtime state.

### Source use

Heuer. Loads cognitive-bias and intelligence-analysis failure modes. CIA Tradecraft Primer. Loads structured analytic technique selection and use conditions. Pherson and Pherson. Loads practical tradecraft: indicators, ACH, key assumptions, diagnosticity, and challenge procedures. Tetlock. Loads judgment calibration, forecasting humility, and expert-performance limits. Kahneman. Loads bias, heuristics, and slow/fast cognition as an error-control vocabulary. A page-family source-use record exists. Final citation strength still depends on page-specific passage review.

### Plan

Plan: identify the region-specific source spine, state what the page can and cannot claim, preserve source-register lineage, and close any remaining source lineage or case-selection gaps before treating the page as final. The project source notes below should be converted into authored subsections only after the page-specific claim limit and source-use decisions are settled.

### Status

Status: native public port complete for upload and navigation; substantive source-project verification remains limited by the source lineage ceiling and the original project governance records. The page separates public presentation from project completion; completion depends on the status and remaining-gap fields.

### Remaining gaps

Remaining gaps: not every regional page has the same mature source lineage foundation as the Russia Desk; thinner pages need page-specific bibliographies and clearer case-selection warrants. The kept Evidence Sections below preserve project substance for close reading; final public prose should be authored from those signals rather than copied mechanically.

## Maintenance Rule

Update this blueprint before strengthening public claims on the website. Do not use word count, module count, or status percentage as a substitute for concrete project substance.
