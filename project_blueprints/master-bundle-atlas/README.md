# Master Bundle Atlas - 152 source and handoff packets

## Public Pitch

**What it is.** Master Bundle Atlas is an evidence-governed catalogue for 152 source, handoff, and project packets.

**Distinctive mechanism.** It routes each packet through artifact identity, evidence state, public claim boundary, portfolio role, and next action so a packet becomes a feature page, source substrate, implementation backlog, private support layer, or archive for stated reasons.

**Why it matters.** It matters because a large project archive only becomes professionally legible when readers can see what each packet is, what it supports, and what cannot yet be claimed from it.

## Blueprint

### Vision

Master Bundle Atlas is an evidence-governed catalogue for 152 source, handoff, and project packets.

### Mechanism

It routes each packet through artifact identity, evidence state, public claim boundary, portfolio role, and next action so a packet becomes a feature page, source substrate, implementation backlog, private support layer, or archive for stated reasons.

### Built components

Verified structural coverage: 152 Master Bundle ZIPs observed; 152 per-ZIP expert briefings found; 0 missing briefings; 5,941 archive entries indexed; 4,447 text-bearing entries decoded; 0 extraction failures. The atlas page preserves the 152 briefing records as navigable public cards while warning that archive coverage is not the same thing as deployed product status.

### Source use

Source authority comes from the per-ZIP expert briefings, archive-entry registry, decoded text-bearing manifest, and portfolio-presence checks. These sources establish artifact existence, evidence state, family, intended use, and claim limit. They do not authorize stronger public claims without page-specific implementation or citation evidence.

### Plan

Use the atlas to promote packets responsibly: first classify by evidence state, then route to feature page, source substrate, implementation backlog, private support, or archive; finally bind any stronger public claim to a named live file, route, source register, or review gate.

### Status

Status: the Master Bundle is structurally audited and navigable in the professional package. It proves an organized 152-packet archive with full briefing coverage, not 152 deployed products.

### Remaining gaps

Remaining gaps: individual packet cards still need page-by-page promotion decisions; older intention-track filters should eventually be reconciled to the evidence-state taxonomy; public feature pages must cite implementation surfaces before any packet is described as live or complete.

## Maintenance Rule

Update this blueprint before strengthening public claims on the website. Do not use word count, module count, or status percentage as a substitute for concrete project substance.
