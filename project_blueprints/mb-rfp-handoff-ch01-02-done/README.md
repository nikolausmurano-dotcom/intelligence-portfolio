## Public Pitch

**What it is.** This is a reusable chapter-analysis pipeline applied to the first two chapters of the Routledge Handbook of Russian Foreign Policy.

**Distinctive mechanism.** The mechanism is method operationalization: name the analytic technique, define its use condition, connect it to a case or judgment problem, record failure modes, and decide whether it is a live method page or a reference packet.

**Why it matters.** It links regional knowledge to analytic method by preserving source lineage, strategic frame, adversary logic, and claim limits for later Russia/security work.

Raw filenames often append Nik_Murano and are not treated as authorship evidence. Catalogue rows are preferred for authors where available. This strengthens page vision, mechanism, source-use, plan, and gap language, but it does not certify passage-level citation support. source-library controlled content is source evidence for internal scholarship and planning; do not redistribute source files or long excerpts in the public package.

### Plan

How this briefing says the vision should be accomplished, translated into a public project route. For each new chapter N: (1) create chapter_XX_*/ folder; (2) produce analysis.md using templates/standard_chapter_schema.md; (3) produce metadata.json; (4) append row to master_tracker.csv; (5) update cumulative_synthesis_seed.md with new themes, overlaps with prior chapters, tensions/contradictions, implications for book-level thesis; (6) update cross-chapter comparison matrix where relevant. Wire or preserve the implementation files: None. The pipeline is documentation-only; the schema file IS the "tool" a human or model uses.

### Status

Completed packet or authority layer

### Remaining gaps

Retain the packet as a completed authority layer, then decide whether it should remain a reference file or become a live room, method page, or public proof object.
