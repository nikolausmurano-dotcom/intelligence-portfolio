# AI Rigor Detection - framework that detects performative rigor in AI-generated analysis; publication status is separated from validation status

## Public Pitch

**What it is.** A rigor-detection framework for AI outputs that look scholarly, structured, or cited while failing to do the underlying evidentiary work.

**Distinctive mechanism.** It separates the appearance of rigor from actual warrant: citation binding, source relevance, reasoning continuity, rubric fit, counterclaim handling, claim strength, Goodhart risk, adversarial-gaming flags, and validation limits are inspected as distinct failure surfaces.

**Why it matters.** It matters because AI-assisted analysis can fail by sounding careful; the project gives reviewers a vocabulary for detecting rigor theater and a path toward testing it against labeled rigorous and performative examples before it becomes accepted judgment.

## Blueprint

### Vision

A rigor-detection framework for AI outputs that look scholarly, structured, or cited while failing to do the underlying evidentiary work.

### Mechanism

It separates the appearance of rigor from actual warrant: citation binding, source relevance, reasoning continuity, rubric fit, counterclaim handling, claim strength, Goodhart risk, adversarial-gaming flags, and validation limits are inspected as distinct failure surfaces.

### Built components

The specification is in place and the validation infrastructure reports measured performance at F1 0.84 under its current labeled test-set conditions. Publication status is not used as proof of the validation claim. Open questions, surfaced by the framework itself: the human calibration pilot is planned but hasn't been executed yet, so the F1 numbers rest on a single coder's labels rather than multi-rater consensus. The Goodhart blocking gap is unresolved — the framework flags Goodhart-style adversarial gaming but doesn't yet block it. And RSC-001 honestly named its own zero recall and meets its own retirement clause; the framework's self-discipline is what surfaced the failure.

### Source use

Source use is organized by role: method authorities, domain evidence, technical background, implementation evidence, and claim limits. The page should show why each source matters; direct quotations and stronger factual claims still require exact passage anchors before publication.

### Plan

From specification-with-measured-validation to calibrated framework with multi-rater authority. Four discrete next studies move the framework forward; each is a contained piece of work, not a re-architecture. Step 01 Multi-rater pilot Lock authoritative threshold cutoffs against consensus ground truth; retire single-coder F1 dependency. Step 02 LLM-as-judge module Close the Goodhart gap — convert adversarial-gaming flags into blocks rather than alerts. Step 03 RSC-001 retirement The single-coder rule retires per its own clause once the multi-rater authority lands. Step 04 Manifest reconciliation Either build the named-but-missing files, or trim the governance manifest to what exists.

### Status

Framework specified and validation claim kept as measured in the page; journal-publication status is not asserted without an official record.

### Remaining gaps

Confirm any journal submission or publication record before using published-article language; close multi-rater calibration and Goodhart-blocking gaps before strengthening the validation claim.
