# Room Master (constitution)

## Public Pitch

**What it is.** Room Master is the MuranoOS constitution for what a room is allowed to be.

**Distinctive mechanism.** It defines room purpose, UI/UX expectations, live state, memory, instruments, export surfaces, and Red Team lossless replacement discipline so future rooms inherit a common architecture instead of improvising their own rules.

**Why it matters.** It matters because MuranoOS can only scale if each room preserves evidence, roles, state, and review obligations in a consistent way.
