# Project Blueprints This directory is the maintainable project-state layer behind the professional website. Each project folder contains: - `README.md`: human-readable vision, mechanism, built state, source use, plan, status, and gaps. - `blueprint.json`: structured version of the same fields for future generators or audits. Governance rules: 1. Do not use word count, section count, progress bars, or module counts as substitutes for substance. 2. Update the blueprint before strengthening public claims on the page. 3. Keep built-state claims tied to components, files, routes, ledgers, datasets, source packets, validators, or review gates. 4. Keep source-use claims below their evidence state. A bibliography is not passage-level claim support. 5. If a project has no source lineage bridge, say so plainly and treat its sources as method and source background until review closes.
