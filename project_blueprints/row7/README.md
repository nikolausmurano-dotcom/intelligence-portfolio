# Discovery Engine — eight-stage cross-disciplinary research-question discovery apparatus

## Public Pitch

**What it is.** A local research-development application with two linked halves: cross-field mechanism discovery and dependency-aware academic workflow.

**Distinctive mechanism.** The discovery side searches for transferable mechanisms from other fields and tests whether they survive context, language, timing, and analogy limits; Observatory, Scanner, Bridge, Cross-Pollination Lab, Assay Lab, Crucible, Incubator, Forge, and the Galaxy Map make those cross-field mechanism relations visible. The academic side carries problem exploration, problem statement, literature review, methodology, analysis, discussion, conclusion, and editing through stage rooms with instruments and standards; upstream changes can mark downstream stages stale, and downstream revisions can force earlier sections back into review.

**Why it matters.** This is the distinctive Nexus claim: it is both an originality engine and a dependency-governed research pipeline, helping a researcher generate non-obvious ideas while keeping the eventual paper aligned from problem statement through final argument.

### Built components

The apparatus is structurally complete. Every methodological-requirement specification across the six build waves passes its tests, every public function in the codebase carries a docstring and type hints, and the discovery workflow runs end-to-end against the gate funnel that narrows the candidate corpus down by an order of magnitude with each successive gate. What blocks publication is not engineering work; it is a framing problem the system has already pre-published against itself. The audit found that the manuscript's original claims — "structural-analogy detector," "computational creativity," "automated discovery" — overclaim what the apparatus actually does, and identified nine fatal gaps where each claim outruns its implementation. The honest reframe is narrower and entirely defensible: the system is a gap-driven cross-disciplinary literature-search engine that uses Thompson-sampling allocation to decide where to look next. That reframe is addressable in the manuscript text; the underlying engineering already does the narrower thing, and does it well. Other open items are equally explicit. About a third of the cognitive-stage scope is open and waiting for the next pass. The Galaxy Map visualizer is the least built subsystem and is currently the bottleneck for any external demo. The discovery pipeline carries a list of high-priority bugs that have to be cleared before any external demo, because while those bugs are open the pipeline produces fingerprint sizes of zero and novelty scores stuck at the midpoint, and that visibly defeats the discovery story.
