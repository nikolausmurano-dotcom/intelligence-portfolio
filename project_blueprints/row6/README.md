# Scholarly Retrieval System — cross-disciplinary research library with a twelve-stage analytical-governance plane

## Public Pitch

**What it is.** A local scholarly retrieval and evidence-review system, not merely a searchable library.

**Distinctive mechanism.** It separates the local corpus plane, retrieval plane, and twelve-stage analytical-governance plane: corpus admission and extraction feed sparse, dense, and semantic indexes; hybrid fusion, metadata filters, domain-aware subcorpora, reranking, and chunk-window expansion shape retrieval; query decomposition, evidence retrieval, seven-stage evidence curation, evidence scoring, analyst drafting, citation validation, claim registration, claim-gap review, follow-up query planning, contradiction review, red-team review, abstention-capable final decision, traceability, release, and source-influence tracking govern what can be said.

**Why it matters.** It matters because RAG quality depends on the whole evidence path, not only the final answer; the system makes sparse and semantic retrieval, evidence support, citation validity, abstention, contradiction, source influence, and refusal-to-answer discipline inspectable before a research claim is trusted. Its current limit is also explicit: corpus ingestion remains the weak plane to repair before using the library as a full defensive-retrieval testbed.

### Built components

The system is live today against roughly four thousand committed books and has been growing the queryable searchable view by an order of magnitude per month, but the corpus admission pipeline is stuck — well over a hundred and forty thousand discovered records sit in a "stable" holding state, blocked by a write-lock contention pattern in the manifest that exhibits as burst-then-die behavior. Until that is resolved, the analytical governance plane is governing over a deliberately under-admitted library. The retrieval indices behind the live four thousand books are large and exercised — sparse and dense indices both, fused by reciprocal-rank fusion — and the analytical-governance discipline above them runs in production end-to-end, refusing to write past its evidence and surfacing contradictions before drawing conclusions. The system is available to a query at any time through three local services, and the discipline is the same on every query. The open questions are equally explicit. There is no bibliographic-identity layer yet, so the system cannot reliably distinguish a work from an edition from a translation from a duplicate. There is no learned reranker — only RRF plus keyword boost — which means relevance ordering is competent but not optimized. And retrieval evaluation has no corpus-derived gold set yet, which means retrieval quality is currently inferred from analytical-stage behavior rather than measured directly. ### Plan

Convert the library system from a thing analysts use into a thing the defensive retrieval-augmented generation research program runs against — with the twelve-stage analytical-governance plane as the page under attack. The order is dictated by what the experiment needs to be valid, not by what is fastest to ship. Step 01 Corpus admission fix Resolve the manifest write-lock contention; drain the stable-state backlog. No inference about governance behavior generalizes against a depleted corpus. Step 02 Poison-injection harness Added to the Corpus plane with full provenance — every injected poisoned token recoverable; experiment reproducible. Step 03 Per-stage instrumentation Each of the twelve analytical-governance stages logs its detection rate against injected poison. The instrumentation is what turns the plane into a measurable defensive structure. Step 04 Matched-pair experiment The disciplined twelve-stage plane against a naive retrieval-augmented generation control, run across the poison classes specified in Row 14. Step 05 Retrieval plane completion Reranker, retrieval evaluation, chunk-window expansion. Retrieval quality is a confound; unmeasured retrieval variance is unmeasured experimental variance. Step 06 Corpus plane completion Bibliographic identity, OCR-quality governance, internal metadata extraction. Corpus integrity is the other confound; an unidentified duplicate is unmeasured corpus variance.
