# Collection Management Analysis

## Public Pitch

**What it is.** A collection-management analysis page for turning intelligence needs into collection priorities.

**Distinctive mechanism.** It links requirements, priority intelligence requirements, source fit, collection gaps, reporting value, and feedback so collection is judged by decision utility rather than activity volume.

**Why it matters.** It shows how an intelligence question becomes a managed collection problem with tradeoffs, not just a list of possible sources.

### Source use

Loads practical tradecraft: indicators, ACH, key assumptions, diagnosticity, and challenge procedures. Tetlock. Loads judgment calibration, forecasting humility, and expert-performance limits. Kahneman. Loads bias, heuristics, and slow/fast cognition as an error-control vocabulary. A page-family source-use record exists. Final citation strength still depends on page-specific passage review.

### Plan

Plan: identify the region-specific source spine, state what the page can and cannot claim, preserve source-register lineage, and close any remaining source lineage or case-selection gaps before treating the page as final. The project source notes below should be converted into authored subsections only after the page-specific claim limit and source-use decisions are settled.

### Status

The page separates public presentation from project completion; completion depends on the status and remaining-gap fields.

### Remaining gaps

The kept Evidence Sections below preserve project substance for close reading; final public prose should be authored from those signals rather than copied mechanically.
