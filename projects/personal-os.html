What this does

A local personal operating system for project development, not a private life-management page.

Record note: project purpose

How it works

Projects move through departments, rooms, subrooms, agents, commands, source records, task state, review surfaces, and release outputs while preserving local memory and upstream/downstream handoffs.

Record note: working mechanism

The local build is grounded in a Tauri / React / Rust / SQLite desktop architecture that can host project workstations rather than only static notes.

Record note: working mechanism

Why it matters

It demonstrates operating-system thinking for intellectual work: serious projects need state, roles, instruments, review loops, continuity, and local software infrastructure, not scattered chats and folders.
