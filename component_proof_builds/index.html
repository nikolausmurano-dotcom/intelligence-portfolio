Component Proof Build System

This layer converts component claims into proof-build obligations. Each component has an artifact to build, an inspection protocol, a source role, a falsification rule, a claim ceiling, and the next concrete capture step. The aim is to make distinctive mechanisms inspectable rather than merely asserted.

Proof Standard

A component is not considered strongly demonstrated because it has persuasive prose. It needs a visible input, an operation, an output, an evidence artifact, a source role, a falsification rule, and a claim ceiling. Where a component remains under-proved, the build packet specifies a new demonstrator to add.
