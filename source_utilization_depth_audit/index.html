Source Depth Audit

Source Utilization Is Not Maxed Out

This layer answers the specific objection that broad book-source coverage is not the same as quote-level, passage-level, and artifact-integrated source use. It classifies every source application by depth and preserves a stricter claim boundary: a machine-routed source may sharpen design, but it is not citation-ready until a human verifies exact passage, location, context, and the artifact change it governs.

Depth Scale

  • L1 Source assigned: a source is attached to a role.
  • L2 File located: a candidate file exists.
  • L3 Machine extracted: text was extracted or scanned.
  • L4 Concept/detail cued: terms, phrase cues, or concept family exist.
  • L5 Source-level machine quote candidate: a short candidate passage exists for the source, but it is not yet matched, verified, and integrated against a specific project claim.
  • L6 Human verified and integrated: exact wording/location/context verified and used in an artifact with before/after proof.

Current release boundary: this package reaches L5 for many rows, but it does not honestly claim L6 completion.
