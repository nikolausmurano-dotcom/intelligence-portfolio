# Proof Build: Red Team Lab — consolidated counter-deception lab across five authority roles

**Project ID.** `red-team-lab`

**Proof tier.** source-backed-proof-build

**Public claim.** A consolidated counter-deception lab for adversarial review of analytical work.

**Distinctive mechanism.** It treats red-teaming as a structured discipline with five authority roles: baseline canon, implementation packet, Prism Observatory, continuation pack, and bootstrap wrapper. The lab preserves six-table SQLite schema logic, canonical JSON project schema, replayable canon, patch-and-validation records, doctrinal core, command-registry deltas, cold-start re-entry, and a lineage appendix so a reader can see which authority supports which claim.

**Use value.** It matters because adversarial review is not a posture; it is a system for preserving assumptions, sources, scope, contradictions, patches, doctrine, replay, and re-entry without forcing the public reader through dozens of internal handoff packages.

**Minimum public demo.** Test one representative case; confirm input (packet name, source files, version signal, scope...) produces output (what the packet is and what decision it can still...) and leaves a visible decision or artifact.

**Claim ceiling.** The page may claim proof-structured components and visible verification targets. It may not claim external validation, production deployment, or completed empirical findings unless the project-specific record states those facts.
