# Proof Build: Personal OS — MuranoOS, a personal operating system, local workbench, and project foundation

**Project ID.** `personal-os`

**Proof tier.** specified-proof-plan-needs-demo

**Public claim.** A local personal operating system for project development, not a private life-management page.

**Distinctive mechanism.** Projects move through departments, rooms, subrooms, agents, commands, source records, task state, review surfaces, and release outputs while preserving local memory and upstream/downstream handoffs. The local build is grounded in a Tauri / React / Rust / SQLite desktop architecture that can host project workstations rather than only static notes.

**Use value.** It demonstrates operating-system thinking for intellectual work: serious projects need state, roles, instruments, review loops, continuity, and local software infrastructure, not scattered chats and folders.

**Minimum public demo.** open one room, create or inspect one action, and show the persisted state record and governing room contract

**Claim ceiling.** The page may claim proof-structured components and visible verification targets. It may not claim external validation, production deployment, or completed empirical findings unless the project-specific record states those facts.

## Flagship Proof Builds

### Room-state operating loop proof

- Claim: MuranoOS is a local operating environment for project work, not a static planning page.
- Input: room contract, task/action, state table, agent or expert assignment, and output packet
- Operation: route the action through local runtime state and write the result back to room history
- Expected output: room/action/state record with source, decision, next action, and export boundary
- Verification check: open one room, create or inspect one action, and show the persisted state record and governing room contract
