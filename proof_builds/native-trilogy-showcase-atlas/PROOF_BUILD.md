# Proof Build: Book Trilogy Atlas

**Project ID.** `native-trilogy-showcase-atlas`

**Proof tier.** specified-proof-plan-needs-demo

**Public claim.** Book Trilogy Atlas is a manuscript-control atlas for a governed three-book inquiry into power, democratic formation, violence, memory, and repair.

**Distinctive mechanism.** It maps seventy-two chapters through canonical IDs, ReaderEdition targets, FinalProof argument backbone, ConstructionKits, PassageTargets, Democratic Dialectic voice material, bridge logic, dependency logic, terminology controls, chapter shape, chapter arcs, title state, visual identity, and rewrite governance across Public Book I, Public Book II, and Public Book III.

**Use value.** It matters because the trilogy's distinctive claim is architectural: power, democratic speech, and political repair have to be managed as one book system without flattening the three books into one voice or confusing public book order with canonical chapter IDs.

**Minimum public demo.** select one chapter from each book and show canonical ID, public book, argument role, visual/title status, and next rewrite task

**Claim ceiling.** The page may claim proof-structured components and visible verification targets. It may not claim external validation, production deployment, or completed empirical findings unless the project-specific record states those facts.

## Flagship Proof Builds

### 72-chapter atlas proof

- Claim: The atlas should show full chapter architecture, public/canonical mapping, title state, visual specificity, and source-governed rewrite workflow.
- Input: 72 chapter IDs, book order, chapter title ledger, visual state, and rewrite-status data
- Operation: bind each chapter to its book, argument role, rewrite target, visual/title state, and next dossier step
- Expected output: chapter-atlas proof record and next rewrite queue
- Verification check: select one chapter from each book and show canonical ID, public book, argument role, visual/title status, and next rewrite task
