# Proof Build: Murano

**Project ID.** `native-murano`

**Proof tier.** specified-proof-plan-needs-demo

**Public claim.** A MuranoOS-linked manuscript command surface for coordinating the Trilogy and related human-agent work.

**Distinctive mechanism.** It connects Tauri / React / Rust / SQLite desktop architecture with rooms, workstations, agents, commands, chapter architecture, reasoning matrices, continuity checks, collision checks, and revision state so manuscript development is treated as a governed operating environment.

**Use value.** It matters because long-form writing and agent-assisted development need local software, memory, roles, dependency checks, and review surfaces rather than isolated drafts.

**Minimum public demo.** trace one UI action to Rust command and SQLite state, then back to visible room output

**Claim ceiling.** The page may claim proof-structured components and visible verification targets. It may not claim external validation, production deployment, or completed empirical findings unless the project-specific record states those facts.

## Flagship Proof Builds

### Tauri / React / Rust / SQLite runtime proof

- Claim: MuranoOS uses a local desktop runtime with UI, command bridge, persistence, agents, and cabinet/department governance.
- Input: React room, Tauri command, Rust backend handler, SQLite table, and room contract
- Operation: invoke one room action through the command bridge and verify persistence
- Expected output: local state mutation, command result, room history, and proof boundary
- Verification check: trace one UI action to Rust command and SQLite state, then back to visible room output
